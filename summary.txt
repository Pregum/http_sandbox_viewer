http_sandbox_viewer は、Flutter アプリ内で HTTP リクエストとレスポンスを視覚的に確認・再実行できる、開発者向けのデバッグ用ライブラリです。

このパッケージの主な目的は、Dio + Retrofit を使った HTTP 通信の内容をアプリ内で簡単に確認できるダッシュボードを提供し、開発中の API デバッグを効率化することです。

主な特徴は以下のとおりです：
- Dio の Interceptor を通じてリクエスト/レスポンスを自動的に記録
- リクエストの詳細（URL、メソッド、ヘッダー、Body）やレスポンスの内容（ステータスコード、ヘッダー、Body）を UI で閲覧可能
- 記録されたリクエストを再編集・再送信可能
- ライブラリ利用者が追加で `flutter_hooks` や `hooks_riverpod` を依存に追加する必要なし
- drift_db_viewer のように、アプリに画面を一つ追加するだけで利用可能

今後は http パッケージや chopper への対応、リクエストのエクスポート、ステータス別のタイムライン表示なども計画しています。
